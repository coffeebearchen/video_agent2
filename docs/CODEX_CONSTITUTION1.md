# 【CODEX_CONSTITUTION_V1｜项目宪法】

This is the core constitution of the project. All future tasks must follow this document.

---

## 一、项目阶段（冻结）

当前项目处于：

**V1 自用稳定版 / 主链稳定化阶段**

当前目标不是做平台，不是做多用户系统，不是做完整商业产品，而是把现有自动视频系统打磨成：

- 可重复运行
- 可回滚
- 可逐步扩展
- 结构清晰
- 主链稳定

当前项目本质定位为：

**结构驱动、时间轴驱动的自动视频生成生产线内核**

---

## 二、主链冻结（最高优先级）

当前唯一有效主链为：

INPUT → SCRIPT → SCENE → ASSETS → TTS → VIDEO

其中：

- `run_pipeline_web.py` = 主链总调度器 / 系统心脏
- 不允许擅自重构主链
- 不允许改变主链执行顺序
- 不允许把主链拆成新架构
- 不允许引入平台化设计

---

## 三、角色定义（必须锁死）

### ChatGPT
负责：
- 边界控制
- 任务拆解
- 风险判断
- 审查与验收标准设计

### Codex / Copilot
角色是：
**受控执行者**
不是架构师，不是产品经理，不是系统设计者。

只负责：
- 读取被授权的最小文件集合
- 在明确边界内修改最少代码
- 交付完整文件
- 严格按任务卡执行

### User
负责：
- 最终判断
- 最终确认
- 冻结阶段目标
- 是否接受变更

---

## 四、开发原则

1. **优先新增模块**
   - 优先新增文件
   - 优先外挂式扩展
   - 不优先修改旧文件

2. **最小侵入原则**
   - 只改必要位置
   - 不做顺手优化
   - 不做无关修复

3. **单任务原则**
   - 一次只做一个任务
   - 一次只改一个明确模块或一个明确接入点
   - 禁止多个目标混做

4. **可回滚原则**
   - 所有改动必须可回滚
   - 删掉新增文件后，系统应尽量恢复原状态
   - 如修改旧文件，必须说明回滚方法

5. **稳定优先原则**
   - 先跑通，再优化
   - 先稳定主链，再增加能力
   - 宁可笨但稳定，不可聪明但不稳

---

## 五、硬禁止项

以下行为一律禁止，除非任务卡明确授权：

- 禁止重构主链
- 禁止主动修改 `run_pipeline_web.py`
- 禁止主动修改 `video_engine.py`
- 禁止主动修改 `scene_planner.py`
- 禁止扫描整个仓库
- 禁止提出平台化架构升级
- 禁止引入数据库
- 禁止引入用户系统
- 禁止引入向量 / embedding / 检索系统
- 禁止引入知识库系统
- 禁止前后端分离
- 禁止服务化改造
- 禁止以“优化”为名改动无关模块
- 禁止把“预留能力”写成“已支持能力”
- 禁止为了一个小任务同时改多个旧文件
- 禁止交付零散 patch 作为主要交付物

---

## 六、读取规则

每次任务只允许读取：

- 当前任务直接相关的最小文件集合
- 默认最多 3～5 个文件
- 若无明确授权，不得读取无关模块
- 不得全仓扫描
- 不得“先看看别的文件再说”

---

## 七、交付规则

默认交付要求：

1. 提供完整文件
2. 标明文件路径
3. 标明本次修改目的
4. 标明运行方式
5. 标明验收方式
6. 标明回滚方式

如果修改旧文件，必须额外说明：

- 为什么非改不可
- 为什么不能通过新增模块实现
- 这次改动是否影响旧主链

---

## 八、当前阶段允许目标

当前阶段允许做的事：

- 稳定主链
- 真实内容测试
- 规则型素材系统迭代
- 素材命中率优化
- 画面质量优化
- 输入层最小增强
- 测试效率提升（前提是不破坏主链）

---

## 九、当前阶段禁止目标

当前阶段明确不做：

- 平台化
- 多用户
- 数据库
- 向量化
- 知识库
- 推荐系统
- AI 智能理解系统重构
- 连续长音频系统
- 前端大改版
- 服务端重构

---
【通用执行规则｜必须遵守】

1. 输出必须是完整可复制粘贴版本，不要省略，不要只给片段。
2. 所有代码必须以“可直接运行”为目标，不允许只写半成品。
3. 新增文件必须放到正确的项目文件夹中，不允许乱放。
4. 不允许创建无关文件、测试脚本、临时脚本、debug 文件。
5. 必须保持项目目录清晰，文件职责明确。
6. 优先保护现有主链，不允许随意改动主链结构。
7. 若需新增模块，必须放入与职责匹配的目录，例如：
   - modules/ 下放功能模块
   - data/current/ 下放当前运行数据
   - output/ 下放输出结果
8. 不允许“顺手优化”无关代码。
9. 如非明确要求，不允许修改多个文件。
10. 输出代码前，先检查应放在哪个文件夹，再开始写。




## 十、核心总结

当前不是“重新设计系统”的阶段，
而是：

**稳定主链 + 小步增强 + 可控扩展 + 可回滚交付**

任何偏离该方向的行为，视为违规。
