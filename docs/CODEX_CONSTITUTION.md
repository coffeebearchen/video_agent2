1. 项目当前阶段是：
V1 自用稳定版 / 主链稳定化阶段
目标是把现有自动视频系统打磨成“可重复运行、可回滚、可逐步扩展”的生产线内核。
当前不是做大平台，不是做多用户系统，不是做前端产品化，不是做数据库，不是做向量化。

2. 主链冻结：
INPUT → SCRIPT → SCENE → ASSETS → TTS → VIDEO
run_pipeline_web.py 是系统主链入口与心脏，不得擅自重构，不得改变顺序，不得把主链拆成新架构。

3. 修改原则：
先新增模块，再最小侵入接入。
优先外壳式扩展，不优先重构已有模块。
任何任务默认只允许修改一个明确模块或一个明确接入点。

4. 禁止项：
禁止主动扫描全仓并给出系统级重构建议。
禁止擅自修改 run_pipeline_web.py、video_engine.py 等核心文件，除非任务卡明确授权。
禁止引入数据库、用户系统、向量检索、知识库、后端服务化、前后端分离。
禁止把“预留能力”伪装成“已支持能力”。

5. 交付原则：
默认交付完整文件，不交付零散 patch。
如需改旧文件，必须解释“为什么非改不可”，并保持最小侵入。
交付必须包含：修改说明、运行方式、验收方式、回滚方式。

6. 读取原则：
每次任务只允许读取“与当前任务直接相关的最小文件集合”。
禁止无边界阅读全仓代码。
若任务不需要某模块，就不要读取该模块。

7. 审计原则：
所有任务必须可验收、可回滚、可隔离。
如果删掉本次新增文件，系统应尽可能恢复到原状态。

8. 行为原则：
Codex 不负责决定项目方向。
Codex 不得把项目从“稳定主链阶段”带向“平台化阶段”。
Codex 的任务是：严格按边界完成单任务，而不是优化一切。
