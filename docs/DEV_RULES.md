【开始指令｜VIDEO_AGENT 固定开发原则】

你正在维护 VIDEO_AGENT 项目。后续所有修改必须严格遵守以下原则：

一、总原则
先走通，再优化
不发散，不重构，不漂移
优先保证现有主链继续可运行
所有新增能力都应以“最小闭环”方式进入系统
二、主链保护原则
不动主链结构，除非明确批准
不随意修改 video_engine 的核心行为
不重排 run_pipeline_user 的执行顺序
不为了新功能破坏旧链路
三、修改方式原则
优先在外层补丁，不内侵核心
优先做最小修改，不做大重构
一个问题只做一个最小修复
新逻辑必须局部、可控、可回退
四、安全原则
默认关闭自动修改能力
patch 可以存在，但不能默认自动执行
不允许未经控制地写回关键数据
所有新增行为都要可解释、可验证
五、路径与结构原则
禁止新增硬编码路径
路径统一走 modules/project_paths.py
不随意改变现有 JSON 结构
保持现有目录结构清晰稳定
六、模块职责原则
一个模块只做一件事
tts_engine_main.py 只负责 TTS
video_engine.py 只负责视频合成
project_paths.py 只负责路径管理
UI 层只负责输入、展示和触发
七、兼容与回退原则
新结构优先，旧结构兜底
能回退就不要硬切
不影响已有非空文本逻辑
空值、缺值场景必须有最小兜底策略
八、输出要求
默认提供完整文件或完整模块
不给零碎 patch，除非明确要求
修改后必须说明：
改了哪个文件
为什么改
是否影响主链
如何验证
【结束指令】
